## Mobile, yes, but in what form

Mobile training covers several realities. Content accessible from a mobile device through a browser, via a responsive site, already covers many use cases without specific development. An installable web application provides additional convenience at moderate cost. A native application, developed for each operating system, offers the most possibilities but at the highest cost. Confusing the need for mobility with the need for a native application leads to over-investment.

## When a native application is justified

A native application is justified by needs that the web does not cover well: offline use, for learners in the field for example, push notifications that sustain engagement and regularity, access to device hardware features, or very frequent use that justifies a place on the home screen. In the absence of these needs, the additional cost of a native application is not justified.

## Thinking usage before tool

The right approach starts from usage. What will learners do on mobile, how frequently, under what conditions, with what benefit for learning. Mobile suits microlearning, spaced reinforcement and quick reference well, less so deep learning. Defining the target usage makes it possible to choose the right form and avoid developing a costly application for a need the web would have covered.

## Key takeaways

- A responsive site or web application suffices in many cases.
- A native application is only justified by specific needs.
- Offline use, notifications, hardware features and high frequency justify the native.
- Confusing the need for mobility with a native application leads to over-investment.
