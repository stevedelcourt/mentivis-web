15:Tb12,Décider de développer une application mobile de formation suppose de peser l'usage réel attendu contre le coût et la complexité d'une application native. Dans beaucoup de cas, un site web responsive ou une application web suffit, à moindre coût. Une application native ne se justifie que par des besoins spécifiques : usage hors ligne, notifications poussées, intégration matérielle, fréquence d'usage élevée. La question n'est pas de suivre une tendance, mais de déterminer si le mobile répond à un besoin d'apprentissage réel et lequel.

## Mobile, oui, mais sous quelle forme

Le mobile en formation recouvre plusieurs réalités. Un contenu accessible depuis un mobile via un navigateur, par un site responsive, couvre déjà beaucoup d'usages sans développement spécifique. Une application web installable apporte un confort supplémentaire à coût modéré. Une application native, développée pour chaque système, offre le plus de possibilités mais au prix le plus élevé. Confondre besoin de mobilité et besoin d'application native conduit à surinvestir.

## Quand l'application native se justifie

Une application native se justifie par des besoins que le web ne couvre pas bien : usage hors connexion, par exemple pour des apprenants sur le terrain, notifications poussées soutenant l'engagement et la régularité, accès à des fonctions matérielles de l'appareil, ou usage très fréquent justifiant une présence sur l'écran d'accueil. En l'absence de ces besoins, le surcoût d'une application native n'est pas justifié.

## Penser l'usage avant l'outil

La bonne démarche part de l'usage. Que feront les apprenants sur mobile, à quelle fréquence, dans quelles conditions, avec quel bénéfice pour l'apprentissage. Le mobile se prête bien au microlearning, au renforcement espacé et à la consultation rapide, moins à l'apprentissage en profondeur. Définir l'usage cible permet de choisir la forme adaptée et d'éviter de développer une application coûteuse pour un besoin que le web aurait couvert.

## Ce qu'il faut retenir

- Un site responsive ou une application web suffit dans beaucoup de cas.
- Une application native ne se justifie que par des besoins spécifiques.
- Hors ligne, notifications, fonctions matérielles et usage fréquent justifient le natif.
- Confondre besoin de mobilité et application native conduit à surinvestir.
- La démarche part de l'usage d'apprentissage réel, pas de la tendance.
